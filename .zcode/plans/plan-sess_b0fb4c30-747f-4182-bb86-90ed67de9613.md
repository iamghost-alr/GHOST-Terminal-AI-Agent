## Goal
Soft-code all hardcoded app/site paths into `config.py`, add a **setup initializer** tool (opens a bundle of apps in one command — Antigravity + ZCode + Brave), and add **full browser control** (open sites, raw URLs, or browser searches) — all driven by the LLM router understanding intent, not keywords.

Nothing else changes. Calculator, weather, notes, time, memory, chat fallback — untouched.

---

## 1. `config.py` — POPULATE (currently empty, this is the soft-coded hub)

Three editable dictionaries + a couple of settings. This is the only file you edit to customize what GHOST can control.

```python
# DESKTOP APPLICATIONS — name you say -> .exe path
APPS = {
    "antigravity": "C:/Users/naman/AppData/Local/Programs/Antigravity IDE/Antigravity IDE.exe",
    "zcode":       "C:/Users/naman/AppData/Local/Programs/ZCode/ZCode.exe",
    "brave":       "C:/Program Files/BraveSoftware/Brave-Browser/Application/brave.exe",
    "calculator":  "C:/Windows/System32/calc.exe",
    "postman":     "C:/Users/naman/AppData/Local/Postman/Postman.exe",
    "mongodb":     "C:/Users/naman/AppData/Local/MongoDBCompass/MongoDBCompass.exe",
}

# WEBSITES — name you say -> URL
SITES = {
    "google": "https://google.com",
    "youtube": "https://youtube.com",
    "github": "https://github.com",
    "claude": "https://claude.com",
    "chatgpt": "https://chatgpt.com",
}

# SETUP BUNDLES — "initialize my setup" opens one of these
SETUPS = {
    "dev":    {"apps": ["antigravity", "zcode", "brave"], "sites": []},
    "coding": {"apps": ["antigravity", "zcode"],           "sites": ["github"]},
}
DEFAULT_SETUP = "dev"
SEARCH_URL = "https://www.google.com/search?q={query}"
```

Add/remove/rename freely — no code changes needed.

## 2. `tools/application.py` — REFACTOR to read `config.APPS`
- Replace the hardcoded `apps` dict with `from config import APPS`.
- Keep `open_app(query)` behavior (strip filler words, fuzzy-match a key, `subprocess.Popen`).
- **Enhancement:** handle multiple apps in one command — `"open brave and calculator"` opens both. Returns a short status line listing what opened.
- Bad path → graceful `"Couldn't find X"` message instead of a raw traceback.

## 3. `tools/sites.py` — REFACTOR into full browser control
- Read `config.SITES` instead of hardcoded dict.
- Three behaviors the LLM can route to (intent-based, not keyword):
  1. **Known site** → `webbrowser.open(config.SITES[name])`.
  2. **Raw URL** (e.g. `"open reddit.com/r/python"`) → detect via regex, open directly.
  3. **Anything else** → open a browser search via `config.SEARCH_URL` (no longer falls back to text `web_search` — that's a separate tool). This is the "whatever I search, open it" behavior.
- Returns a short confirmation of what opened.

## 4. `tools/setup.py` — NEW FILE (the setup initializer)
- `run_setup(query)` → reads `config.SETUPS`, picks the bundle (default `config.DEFAULT_SETUP`), launches every app in `apps[]` via `subprocess.Popen` and every site in `sites[]` via `webbrowser.open`.
- Returns a status line: `"Setup 'dev' initialized: opened Antigravity, ZCode, Brave."`
- Unknown bundle name → `subprocess.Popen` message listing the available bundles.

## 5. `router/llm_router.py` — add `setup` to the brain
- Add `"setup"` to `VALID_TOOLS`.
- Add a routing-prompt line: `- setup : launch a predefined bundle of apps/sites ("initialize my setup", "start my dev environment", "open my coding setup")`.

## 6. `router/tool_router.py` — wire `setup` into the dispatch table
- Import `run_setup` from `tools.setup`.
- Add `from tools.setup import run_setup` + a `_handle_setup(query)` handler + `"setup": _handle_setup` entry in `_DISPATCH`.

## Resulting runtime behavior (all keyword-free, LLM-routed)

| You say | LLM routes to | What happens |
|---------|---------------|--------------|
| "initialize my setup" | `setup` | opens Antigravity + ZCode + Brave |
| "start my coding environment" | `setup` | opens the `coding` bundle |
| "open zcode" | `application` | launches ZCode |
| "open brave and calculator" | `application` | launches both |
| "open github" | `sites` | opens github.com in browser |
| "open stackoverflow.com/q/123" | `sites` | opens that exact URL |
| "open best mechanical keyboards" | `sites` | opens a Google search in browser |

Every path lives in `config.py` — edit one file to rebind a name, add an app, change a bundle, or swap the search engine.

## What stays untouched
Calculator, weather, notes, time/date, memory, web_search, the chat-API fallback, UI, voice pipeline, `.env`, `requirements.txt`. The refactor is strictly additive to the routing layer + a new config hub.
